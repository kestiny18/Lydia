# Contributing to Lydia

Thanks for helping improve Lydia.

## What We Welcome
- Bug reports and repro steps
- Feature requests and design feedback
- Documentation fixes and clarifications
- Code contributions (small, reviewable PRs)

## Development Workflow
1. Fork the repo and create a branch.
2. Install dependencies: `pnpm install`
3. Run tests before opening a PR: `pnpm test`

## Pull Request Guidelines
- Keep changes small and focused.
- Update docs when behavior changes.
- Include tests when adding or changing core logic.
- Explain the motivation and risk in the PR description.

## Security
If you find a security issue, please avoid filing a public issue. Open a private report instead.
